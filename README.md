# Band Sessions

Десктоп-приложение для двух музыкантов, которые работают над одними и теми же проектами Logic Pro / GarageBand через общую папку на Яндекс.Диске.

**Проблема:** оба участника могут открыть один и тот же проект одновременно и случайно перезаписать правки друг друга. Без координации приходится каждый раз договариваться в Telegram — "ты закончил? можно брать?".

**Решение:** приложение добавляет лок-механизм поверх обычной синхронизации папки. Никакого сервера — стейт хранится в JSON-файлах внутри той же папки на Диске.

---

## Возможности

- **Список проектов** с актуальным статусом: свободно / кто сейчас работает и как давно
- **Лок на трек** — "начать работу" блокирует проект для партнёра, пока ты не завершишь
- **Несколько вариантов** — в одной папке трека может быть несколько `.logicx`/`.band` файлов (основной + экспериментальный и т.д.), каждый открывается из приложения
- **Открыть в DAW** — прямо из карточки трека, без поиска файла в Finder
- **История сессий** — кто, когда и что сделал, с комментариями
- **Реалтайм обновления** — изменения партнёра появляются мгновенно (file watcher), не нужно вручную обновлять
- **Авто-версия при сохранении** — если ты держишь лок и сохраняешь файл в Logic/GarageBand, версия инкрементируется автоматически

---

## Структура папок на Яндекс.Диске

Приложение ожидает (и создаёт при первом запуске) следующую структуру:

```
/Группа/                        ← общая папка группы на Яндекс.Диске
  projects/                     ← все треки живут здесь
    название-трека/             ← одна папка = один трек
      main.logicx               ← рабочий файл (любое имя)
      experimental.logicx       ← опциональный второй вариант
      .session.json             ← стейт трека (создаётся приложением)
      history.json              ← история сессий (создаётся приложением)
    другой-трек/
      ...
  shared/                       ← общие референсы, приложение не трогает
```

Приложение сканирует `projects/` и находит все `.logicx` и `.band` файлы. Папка без `.session.json` показывается как "неинициализированная" — её можно инициализировать прямо из приложения.

## Бизнес-логика

### Флоу работы над треком

1. Открываешь приложение → видишь список треков со статусами
2. Нажимаешь **"начать работу"** → трек блокируется, партнёр видит "занято / твоё имя"
3. Открываешь нужный вариант в Logic/GarageBand через кнопку **"открыть"**
4. Работаешь. Каждое сохранение в DAW автоматически обновляет версию (пока лок твой)
5. Нажимаешь **"завершить"** → опционально оставляешь комментарий ("свёл барабаны"), лок снимается, версия фиксируется
6. Партнёр видит трек свободным и может брать

### Лок

Лок — кооперативный сигнал, не техническая блокировка на уровне файловой системы. Приложение не мешает партнёру открыть файл напрямую через Finder — оно лишь показывает, что кто-то сейчас работает. На практике этого достаточно для двух людей, которые и так общаются.

Если участник А уронил компьютер и не может снять лок — участник Б может воспользоваться **"сбросить лок"** (не инкрементирует версию).

### Варианты внутри трека

Все `.logicx` и `.band` файлы в папке трека считаются вариантами. Лок распространяется на всю папку целиком — нельзя одновременно работать над двумя вариантами одного трека. Это сознательное ограничение: если нужна параллельная работа, создайте отдельные треки.

### Синхронизация

Приложение не синхронизирует файлы само — это делает Яндекс.Диск. Приложение только:
- Читает/пишет `.session.json` и `history.json` в папке трека
- Следит за изменениями этих файлов через file watcher
- При изменении `.session.json` (партнёр взял/снял лок) — обновляет UI в реальном времени

## Настройка

### Требования

- macOS (приложение использует системный Safari WebView)
- Node.js 20+ (`nvm use` если используешь nvm)
- Rust (`rustup` — устанавливается один раз)
- Яндекс.Диск с общей папкой для группы

### Запуск в режиме разработки

```bash
nvm use          # переключает на Node 20 (из .nvmrc)
npm install
npm run tauri dev
```

### Первый запуск (пользователь)

1. Открыть приложение
2. Ввести своё имя (будет отображаться партнёру)
3. Выбрать папку группы на Яндекс.Диске — приложение создаст `projects/` и `shared/` внутри
4. Добавить первый трек через **"+ новый проект"** или положить папку с файлами в `projects/` вручную и нажать "инициализировать"

### Сборка финального приложения

```bash
npm run tauri build
```

Даёт `.app` и `.dmg` в `src-tauri/target/release/bundle/`. Для передачи партнёру без подписи разработчика — открывать через правый клик → "Открыть" (обход Gatekeeper).
