# Band Sessions

Десктоп-приложение для двух музыкантов, которые работают над одними и теми же проектами Logic Pro / GarageBand через общую папку на Яндекс.Диске.

**Проблема:** оба участника могут открыть один и тот же проект одновременно и случайно перезаписать правки друг друга.

**Решение:** приложение отслеживает кто сейчас работает над каждым треком — автоматически, без кнопок. Стейт хранится в JSON-файлах внутри той же папки на Диске, сервер не нужен.

---

## Как это работает

Никаких ручных действий не требуется. Просто открываешь проект в GarageBand или Logic Pro как обычно:

1. **Открыл DAW** → приложение видит изменение файлов и автоматически помечает трек как занятый. Партнёр сразу видит **"kaya работает прямо сейчас"**
2. **Сохранил** → версия трека инкрементируется, в историю пишется "сохранил main.band"
3. **Закрыл DAW** → приложение снимает пометку, трек снова свободен

Партнёр видит обновления в реальном времени через file watcher — Яндекс.Диск синхронизирует небольшой `.session.json` файл, watcher его подхватывает.

---

## Возможности

- **Список треков** с актуальным статусом: свободно / кто работает
- **Варианты** — в одной папке трека может быть несколько `.logicx`/`.band` файлов, каждый открывается из приложения одной кнопкой
- **История** — кто, когда и что сохранял, с именем файла
- **Заметки** — текстовые сообщения к каждому треку: последняя видна на карточке, все — на отдельной странице. Можно добавлять, редактировать, удалять
- **Реалтайм** — обновления приходят без ручного refresh
- **Новый проект** — создать папку трека прямо из приложения

## Структура папок на Яндекс.Диске

```
/Группа/                        ← общая папка группы
  projects/                     ← создаётся кнопкой в приложении или вручную
    sunset/                     ← одна папка = один трек
      main.logicx               ← рабочий файл (любое имя)
      experimental.logicx       ← опциональный второй вариант
      .session.json             ← стейт трека (создаётся приложением)
      history.json              ← история версий (создаётся приложением)
      notes.json                ← заметки участников (создаётся приложением)
    night-drive/
      night-drive.band
      .session.json
      history.json
      notes.json
  shared/                       ← общие файлы, приложение не трогает
```

## Настройка

### Требования

- macOS или Windows
- Один из поддерживаемых DAW (см. ниже)
- Яндекс.Диск с общей папкой (расшарить через веб-интерфейс Диска)

### Запуск в фоне

Приложение работает как менюбар-приложение: крестик скрывает окно, процесс остаётся живым. Открыть снова — клик по иконке в трее (macOS — строка меню, Windows — системный трей). Выйти — через меню иконки "Выйти".

Пока приложение запущено в фоне, оно присылает системные уведомления когда другой пользователь начинает или заканчивает работу над треком.

### Первый запуск

1. Открыть приложение
2. Ввести своё имя — оно будет отображаться у другого участника
3. Выбрать папку группы на Яндекс.Диске
4. Нажать **"создать структуру папок"** — создаст `projects/` и `shared/`
5. Добавить первый трек через кнопку **+** в хедере или положить папку с проектом в `projects/` вручную и нажать "инициализировать" на карточке

### Для разработчика

```bash
nvm use          # Node 24 (из .nvmrc)
npm install
npm run tauri dev
```

```bash
npm run tauri build   # → .app и .dmg в src-tauri/target/release/bundle/
```

Партнёру без подписи разработчика: открывать через правый клик → "Открыть" (обход Gatekeeper).

## Поддерживаемые DAW

| DAW | Форматы |
|---|---|
| Logic Pro / GarageBand | `.logicx`, `.band` |
| Ableton Live | `.als` |
| Reaper | `.rpp` |
| Pro Tools | `.ptx`, `.ptf` |
| Cubase / Nuendo | `.cpr` |
| Bitwig Studio | `.bwproject` |
| FL Studio | `.flp` |
| Digital Performer | `.dpdoc` |

Приложение автоматически определяет какой DAW запущен и снимает лок при его закрытии.
